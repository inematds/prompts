📝 Markdown como Padrao

Markdown e o formato mais versatil para saidas de LLMs. Ele e legivel como texto puro, renderiza bonito em qualquer plataforma e oferece estrutura suficiente para a maioria dos casos.

📋 Elementos Essenciais

## Headers — para secoes e hierarquia

- Listas — para itens sem ordem

1. Numeradas — para passos sequenciais

**Negrito** — para destaques

| Tabelas | — para dados comparativos

```codigo``` — para trechos tecnicos

💡 Quando Usar Markdown

Use Markdown quando a saida sera lida por humanos: relatorios, resumos, analises, emails, documentacao. E o formato padrao para a maioria das interacoes com LLMs.

2

🔧 JSON Estruturado

JSON e o formato ideal quando a saida sera processada por outro sistema — APIs, automacoes, bancos de dados. Voce define o schema e o modelo preenche.

📋 Como Pedir JSON

Formato de saida: JSON com a seguinte estrutura:

{

"titulo": "string, max 50 caracteres",

"resumo": "string, max 100 palavras",

"tags": ["array de 3-5 strings"],

"prioridade": "alta | media | baixa",

"acionavel": true/false

}

📊 Boas Praticas JSON

  • Defina tipos: string, number, boolean, array — evita surpresas
  • Marque obrigatorios: "campo (obrigatorio)" vs "campo (opcional, default: null)"
  • De um exemplo: Incluir um JSON de exemplo reduz erros em 90%
  • Peca "somente JSON": "Responda APENAS com o JSON, sem texto antes ou depois"
3

📊 Tabelas e Comparativos

Tabelas sao ideais para comparacoes lado a lado, listas estruturadas e dados que precisam de multiplas dimensoes. Defina colunas, limites por celula e criterios de preenchimento.

📋 Como Pedir Tabelas

Formato: Tabela Markdown com as colunas:

| Ferramenta | Preco | Ponto Forte | Limitacao |

|------------|-------|-------------|-----------|

Restricoes:

- Exatamente 5 linhas

- Maximo 15 palavras por celula

- Ordenar por preco (menor → maior)

💡 Tabela vs Lista vs JSON

Tabela: para comparar itens em multiplas dimensoes. Lista: para itens sequenciais ou sem comparacao. JSON: para dados que serao processados por codigo. Escolha pelo consumidor da saida.

4

📐 Campos Obrigatorios e Opcionais

Definir quais campos a resposta DEVE ter e quais sao opcionais garante completude sem rigidez excessiva. E como definir o schema de um formulario.

Campos Obrigatorios

  • ✓Titulo (sempre presente)
  • ✓Descricao (1-2 frases)
  • ✓Classificacao (categoria)
  • ✓Acao recomendada

Campos Opcionais

  • ~Exemplo (se aplicavel)
  • ~Fonte/referencia
  • ~Nivel de confianca
  • ~Notas adicionais
5

🎨 Tom e Estilo na Saida

O tom define como o modelo se comunica. O mesmo conteudo pode ser entregue de formas radicalmente diferentes conforme o tom especificado.

🎭 Espectro de Tom

  • •Formal/tecnico: Relatorios executivos, documentacao tecnica, pareceres
  • •Profissional acessivel: Posts LinkedIn, emails corporativos, apresentacoes
  • •Didatico: Tutoriais, explicacoes, material de treinamento
  • •Casual/conversacional: Redes sociais, blogs, comunicacao interna
  • •Persuasivo: Copy de vendas, landing pages, CTAs

💡 Como Especificar Tom

Nao basta dizer "tom profissional". Seja especifico: "Tom profissional mas acessivel. Sem jargao tecnico. Frases curtas (max 20 palavras). Use 'voce' em vez de 'o usuario'. Inclua exemplos praticos."

🧪 Exercicio: Formatos na Pratica

Pegue uma unica tarefa e peca a saida em 3 formatos diferentes: Markdown, JSON e tabela. Compare os resultados e entenda quando cada formato funciona melhor.

📋 Tarefa do Exercicio

Tarefa: Analisar 3 ferramentas de IA para criacao de conteudo

Contexto: Para equipe de marketing de PME, orcamento limitado

Versao A: Formato Markdown (headers + listas)

Versao B: Formato JSON (array de objetos)

Versao C: Formato Tabela (colunas comparativas)

Compare: qual formato e mais legivel? Qual e mais processavel? Qual seu publico prefere?
